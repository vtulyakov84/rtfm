### Аутентификация
Это процесс, при котором клиент (например, программа для считывания показаний или головная система) и сервер (например, сам счетчик) доказывают друг другу свою подлинность. Простыми словами: это предъявление паспорта для подтверждения, что вы — тот, за кого себя выдаете. В DLMS/COSEM для этого используется несколько уровней:

`Lowest Level Security` (`LLS`) — низший уровень: По сути, отсутствие защиты, подразумевает публичный доступ к ограниченному набору данных.

`Low Level Security` (`LLS`) — низкий уровень: Клиент должен предоставить пароль для аутентификации.

`High Level Security` (`HLS`) — высокий уровень: Более надежный, многоэтапный (4 прохода) процесс, когда клиент и сервер обмениваются шифрованными запросами для взаимной проверки.


В протоколе DLMS/COSEM **аутентификация** служит для проверки личности, а **авторизация** определяет разрешенные действия. Оба механизма являются ключевыми для построения безопасной системы, особенно в таких критических областях, как «умные» сети электроснабжения (Smart Grid).

`Association Association` (Прикладная Ассоциация, AA — Application Association): Логическое соединение, устанавливающееся между клиентом и сервером для обмена данными. В рамках этого соединения и происходят процессы аутентификации и авторизации.

`Application Context` (Прикладной контекст): Набор правил для Application Association (AA), определяющий, используются ли шифрование и имена объектов. Возможные контексты:

* `LN`

* `SN`

* `LN_WITH_CIPHERING`

* `SN_WITH_CIPHERING`

`Ciphering` (Шифрование): Преобразование данных в нечитаемый вид для обеспечения конфиденциальности.
В DLMS/COSEM шифрование может применяться:

* *Только аутентификация*: Данные не шифруются, но добавляется специальный тег (MAC), подтверждающий их подлинность.

* *Только шифрование*: Данные шифруются, но не аутентифицируются.

* *Аутентификация и шифрование*: Данные и шифруются, и аутентифицируются для максимальной защиты.

`Access Security` (Безопасность доступа): Относится к аутентификации и авторизации при установлении соединения (Association).

`Transport Security` (Безопасность транспорта): Относится к шифрованию (ciphering) передаваемых данных.

`Key Management` (Управление ключами): Процесс безопасной генерации, распределения, хранения и обновления криптографических ключей, используемых для аутентификации и шифрования.

`Role` (Роль): Заранее определенный набор прав доступа, который может быть назначен прошедшему аутентификацию клиенту.
