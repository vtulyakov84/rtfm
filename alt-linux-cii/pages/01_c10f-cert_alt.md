# Использование сертифицированного образа c10f-cert/alt на объектах КИИ

> Апробация материала выполнена на ОС ALT Linux, 6.1.115-un-def-alt1

*Время чтения: ~3 минуты*

При создании систем на объектах **критической информационной инфраструктуры (КИИ)** требования к безопасности и импортозамещению значительно выше, чем для обычного корпоративного ПО.

Согласно официальной документации ALT Linux, реестр контейнеров `registry.altlinux.org` предназначен для хранения образов в формате OCI.
Ветка `c10f/` содержит образы на базе репозитория для дистрибутива **"Альт СП"** - специализированной платформы, разработанной с учётом требований для защищённых систем.

**Ключевые требования к образам для КИИ**

1. Полный контроль состава используемых пакетов.

2. Регулярное обновление образов для выявления и закрытия уязвимостей (все образы в реестре Base ALT автоматически проверяются на наличие уязвимостей).

Образ `registry.altlinux.org/c10f/alt` предоставляет собой чистую, сертифицируемую основу, позволяющую вписаться в требования к доверенной среде.

В статье используется сертифицированный образ:
`registry.altlinux.org/c10f-cert/alt`.

Полный список доступных образов:
https://registry.altlinux.org/explore?search=c10f-cert

## 1. Подготовка инструментальных средств
Для работы с образом `c10f-cert/alt` необходимо установить `docker-engine `и назначить права пользователю для запуска команд docker без `sudo`:

```bash
# Установка Docker
sudo apt-get install docker-engine -y

# Добавление пользователя в группу docker
sudo usermod -aG docker ${USER}
```
**Важно:** после добавления пользователя в группу `docker` необходимо перелогиниться в ОС, чтобы изменения вступили в силу.

## 2. Запуск контейнера на базе образа c10f-cert/alt
### 2.1 Запуск контейнера
```bash
docker run -it registry.altlinux.org/c10f-cert/alt:latest /bin/bash
```
* Если образа нет в локальном репозитории, он будет автоматически скачан из `registry.altlinux.org`.

* После успешного подключения приглашение командной строки сменится на:

```bash
[root@... /]#
```

### 2.2 Выход из контейнера
```bash
[root@... /]# exit
```
После этого вы вернётесь в хостовую систему:

```bash
[user@... docker_builds]$
```

### 2.3 Работа с образами локально
Используемый образ сохраняется в локальном хранилище хоста:

```bash
# Просмотр локального хранилища образов
docker images
```

При последующих запусках контейнера используется образ из локального хранилища (повторная загрузка не требуется).

## 3. Работа внутри контейнера
После запуска контейнера с подключением к bash:

```bash
docker run -it registry.altlinux.org/c10f-cert/alt:latest /bin/bash
```

### 3.1 Обновление списка пакетов

```bash
# Выполняем обновление списка пакетов
apt-get update
```

### 3.2 Поиск и установка пакетов
```bash
# Поиск пакета <package-name> в репозитории
apt-cache search <имя-разыкиваемого-пакета>

# Установка пакета <package-name>
apt-get install <имя-устанавливаемого-пакета>
```

### 3.3 Очистка кэша пакетов
```bash
apt-get clean
```

#### Преимущество описанного подхода
Работа с командной строкой контейнера позволяет отработать необходимые команды перед тем, как зафиксировать их в Dockerfile.

### Важное замечание
При выходе из контейнера и повторном запуске образа его состояние будет первоначальным.
Образ, на основе которого запускается контейнер, не сохраняет изменения, выполненные в сеансе командной строки.

## 4. Создание Docker-образа с изменённым состоянием
Чтобы получить контейнер с предустановленным набором пакетов (определённого состояния), который будет воспроизводиться при каждом запуске, необходимо:

1. Создать Dockerfile

2. Указать в нём базовый образ
`c10f-cert/alt:latest`

3. Описать необходимые слои (установка пакетов, настройки)

4. Собрать новый образ командой `docker build`

### Пример минимального Dockerfile:

```dockerfile
FROM registry.altlinux.org/c10f-cert/alt:latest

RUN apt-get update && \
    apt-get install -y
        <имя-пакета-1> \
        <имя-пакета-2> \
        ... \
        <имя-пакета-N> && \
    apt-get clean
```

После сборки мы получаем собственный образ - полностью контролируемый, повторяемый и готовый для использования в вычислительной среде КИИ.

## Итог
Использование сертифицированного образа `c10f-cert/alt` позволяет:

* Соблюдать требования к доверенной среде на объектах КИИ

* Контролировать состав пакетов

* Опираться на платформу «Альт СП» с регулярными обновлениями безопасности

* Строить повторяемые контейнеры с нужным состоянием через Dockerfile
