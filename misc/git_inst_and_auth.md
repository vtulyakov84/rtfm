# Git. Установка и настройка авторизации на GitHub

Пошаговая инструкция по установке Git на Linux и настройке авторизации на GitHub для выполнения коммитов.

### Шаг 1. Установка Git

```bash
# Обновляем список репозиториев пакетов
$ sudo apt-get update

# Выполняем установку git
$ sudo apt-get install git -y
```

### Шаг 2. Начальная настройка Git

После установки Git необходимо указать данные о себе, которые будут записываться в каждый коммит.

```bash
$ git config --global user.name "Your Name"

$ git config --global user.email "your_email@example.com"
```

### Шаг 3. Настройка авторизации на GitHub

Современным и безопасный способ авторизации на GitHub является использование SSH-ключей, что позволит не вводить логин и пароль при каждом действии с удаленным репозиторием.

```bash
# 1. Генерация SSH-ключа (в случае его отсутствия)
$ ssh-keygen -t ed25519 -C "your_email@example.com"
```

* `-C "your_email@example.com"`, комментарий для идентификации ключа.

* При запросе места хранения файла нажимайте `Enter`, чтобы оставить путь по умолчанию (`~/.ssh/id_ed25519`).

* При необходимости можно установить пароль для ключа (`passphrase`).

```bash
# 2. Добавление приватного ключа в фоновую программу-менеджер
eval "$(ssh-agent -s)"

ssh-add ~/.ssh/id_ed25519
```

```bash
# Отобразить содержимое файла ключа
cat ~/.ssh/id_ed25519.pub

```

Скопируйте содержимое файла ключа, весь текст, начиная с `ssh-ed25519` и заканчивая указанным `email`.

4. Добавление ключа в аккаунт на GitHub:

* Авторизоваться на GitHub.

* Нажмите на своё фото в правом верхнем углу → выберите **Settings**.

* В левом меню выберите **SSH and GPG keys**.

* Нажмите зелёную кнопку **New SSH key**.

* В поле `Title` введите понятное для вас название.

* В поле `Key` вставьте скопированный публичный ключ.

* Нажмите **Add SSH key**.

### Шаг 4. Проверка подключения

```bash
# Выполняем проверку подключения к github.com
$ ssh -T git@github.com
```

Если вы всё сделали верно, вы увидите сообщение:
`Hi <your-username>! You've successfully authenticated...`

### Шаг 5. Работа с коммитами

1. **Клонируйте ваш репозиторий** (используйте **SSH-адрес**, который можно скопировать на странице репозитория GitHub, нажав кнопку "Code" и выбрав вкладку "SSH").

```bash
# Клонировать репозиторий на хостовую систему
$ git clone git@github.com:username/your-repo-name.git

# Выполняем переход в каталог проекта
$ cd your-repo-name.git
```

2. **Внести изменения** в файлы внутри папки репозитория.

3. Добавление файлов в коммит:

```bash
$ git add .  # Добавляет все изменённые файлы
# или
$ git add имя_файла # Добавляет конкретный файл
```

4. Создание коммита:

```bash
$ git commit -m "Ваше сообщение о том, что было сделано"
```

5. Отправка коммита на GitHub:
```bash
$ git push origin main
```

(Название основной ветки может быть main или master, уточните это в вашем репозитории).
